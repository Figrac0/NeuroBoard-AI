# ❓ Q&A банк (короткие ответы)

- **flat-vector vs temporal-KG:** flat — это эмбеддинги и косинус, времени и связей не знает, устаревший
  факт не отличит. Temporal-KG (Zep/Graphiti) — узлы и рёбра с интервалами валидности, supersession,
  запрос «на момент времени». Агенту, который живёт во времени, нужен temporal.
- **worktree / shared list / lock:** свой worktree на агента — это изоляция; общий QUEUE.md — это список;
  claim.sh с атомарным mkdir-локом — это очередь, задачу правит один.
- **open+harness > frontier-naive:** verify, рефайн и ретраи делают слабую модель надёжной; у naive такой
  страховки просто нет.
- **агент перестраивает план:** план — это функция от состояния (LLM или правила); а внутри прогона
  verify→refine→escalate меняет ход по результату каждого шага.
- **Spec-Driven:** OpenSpec — для существующих проектов (дельты, Propose→Apply→Archive), Spec Kit — для
  проектов с нуля.
- **escalation:** N рефайнов → модель посильнее → needs-human; всё пишется в evidence и откатывается
  снапшотом.
- **persistence:** tmux плюс resurrect плюс continuum (`@continuum-restore on`, `@resurrect-processes`).
- **eval плоский 1.00:** это потолок на простой задаче и сильной модели; ценность harness — на
  слабых/сложных задачах и в колонках рефайны/эскалации, плюс живой цикл.
- **опыт Spec-Driven >3 мес:** не ври — «метод освоил и применил здесь»; показываешь владение, а не стаж.
